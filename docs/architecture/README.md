# Architecture levels

Status: Accepted

AgentMesh 使用四层设计深度。这里的 L0–L3 是“架构文档层级”，与任务执行复杂度分级中的 L0–L4 无关。

| Level | 关注点 | 主要读者 | 典型产物 |
|---|---|---|---|
| L0 | 为什么建设、系统边界、参与者、核心能力、重要约束 | 产品、架构、开源贡献者 | 系统上下文、领域边界、端到端场景 |
| L1 | 哪些可部署容器组成系统、如何通信、数据归谁 | 架构、平台与运维团队 | 容器图、信任边界、数据流、部署拓扑 |
| L2 | 一个容器内部由哪些模块组成、模块契约是什么 | 模块维护者 | 组件图、状态机、接口和失败模型 |
| L3 | 如何实现 | 开发者 | API、Schema、算法、配置和测试设计 |

## Progression rules

1. L0 先确认系统做什么与不做什么。
2. L1 只为已确认的能力分配容器和数据所有权。
3. L2 每次聚焦一个 L1 容器，避免全系统同时深入。
4. L3 在对应的 L2 契约稳定后展开。
5. 涉及多个容器且难以逆转的决策通过 ADR 记录。
